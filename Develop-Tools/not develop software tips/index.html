PicGo 上传图片问题​

当使用PicGo(app) 或者 PicList时，出现如下报错 "message": "getaddrinfo ENOTFOUND smms",

只需切换到PicGo-Core(command line)， 请修改 ～/.picgo/config.json文件，

{
  "picBed": {
    "uploader": "smms",
    "current": "smms",
	"smms": {
      "token": "put your token here"
    }
  },
  "picgoPlugins": {}
}

在vscode中​

使用typora中安装的PicGo-Core​

  1. 在vscode中安装PicGo插件
  2. 在vscode中设置PicGo的Picgo: Config Path为 C:\Users\xxx\.picgo\config.json

使用npm安装的picgo​

# 查看 pnpm 全局安装路径
pnpm root -g

# 查找 picgo 位置
pnpm list -g picgo

  1. 在vscode中安装PicGo插件
  2. 在vscode中设置PicGo的Picgo: Config Path为 C:\Users\xxx\AppData\Local\pnpm\global\5\node_modules\picgo\config.json

U30Air SMB共享问题​

解决办法（启用来宾 SMB 登录）,通过本地组策略（适合 Win10 专业版/企业版）

  1. Win + R → 输入 gpedit.msc → 回车。

  2. 路径：

    计算机配置 → 管理模板 → 网络 → Lanman 工作站

  3. 找到策略：

    启用不安全的来宾登录 (Enable insecure guest logons)

  4. 双击 → 选择 已启用 → 确定。

  5. 运行 gpupdate /force 刷新策略。

win10系统拷贝文件或者删除文件问题​

问题：剩余时间：正在计算
处理方案: 使用CMD管理员模式执行 ，然后重启电脑。

sfc /scannow

执行后的结果：Windows 资源保护找到了损坏文件并成功修复了它们。
对于联机修复，位于 windir\Logs\CBS\CBS.log 的 CBS 日志文件中
有详细信息。例如 C:\Windows\Logs\CBS\CBS.log。对于脱机修复，
/OFFLOGFILE 标记提供的日志文件中有详细信息。

如果还是没有效果，使用WiseForce Deleter删除文件夹。

win10禁用更新​

gpedit.msc

在“本地组策略编辑器”窗口的左侧面板中，请根据以下路径导航到Windows Update文件夹：“计算机配置”>“管理员模板”>“Windows组件”>“Windows更新”。 更改“删除使用所有windonws更新功能的访问权限”为启用。 更改“配置自动更新为”为禁用。
