onClickä¼ å€¼å‡½æ•°åoråŒ¿åå‡½æ•°â€‹

Version 1 (onClick={getRecipe})

Youâ€™re giving React a function reference. React will call getRecipe() only when the button is clicked.

This is the simplest and most efficient form â€” no new function is created on every render.

Version 2 (onClick={() => getRecipe()})

Youâ€™re creating a new anonymous function on each render that calls getRecipe() inside it.

It still works â€” React will call your wrapper function on click, which in turn calls getRecipe() â€” but itâ€™s one extra layer of indirection and (very slightly) less efficient.

import React, { useState } from "react";
import "./index.css";
import Die from "./Die";
import { nanoid } from "nanoid/non-secure";

export interface DiceProp {
  value: number;
  isHeld: boolean;
  id: string;
}

const App: React.FC = () => {
  const generateDice = () =>
    Array.from({ length: 10 }, () => ({
      value: Math.ceil(Math.random() * 6),
      isHeld: false,
      id: nanoid(),

  ));

    setDice(generateDice());
  };

  return (
    <main>
      <div className="dice-container">{diceElements}</div>

      </button>

import { useEffect, useState } from "react";

export default function WindowTracker() {
  const [windowWidth, setWindownWidth] = useState<number>(window.innerWidth);
  useEffect(() => {
    const watchWindowWidth = () => {
      console.log("resized!");
      setWindownWidth(window.innerWidth);
    };
    window.addEventListener("resize", watchWindowWidth);
    // cleaning up what creating before
    return () => {

    };
  }, []);
  return <h1>Window width: {windowWidth}</h1>;
}

lazy Initialization for useStateâ€‹

1ï¸âƒ£ Lazy initialization: useState(() => generateDice())

  • Here, you pass a function to useState.
  • React will call that function once â€” only on the initial render.
  • The return value of the function becomes the initial state.
  • After that, React never calls the function again, because it already has the state stored internally.
  • Benefit: Useful when computing the initial state is expensive.

Explain:

  1. Component mounts â†’ React calls () => generateDice().
  2. The return value (DiceProp[]) is stored as the state.
  3. Component re-renders â†’ React reuses the stored state, function is never called again.

2ï¸âƒ£ Non-lazy way: const [dice, setDice] = useState(generateDice());

  • Here, you call generateDice() immediately and pass its return value to useState.
  • This happens every render of the component, but only the first value matters for initialization.

Explain:

  1. Component renders â†’ generateDice() runs â†’ returns array â†’ React stores as state.
  2. Component re-renders later â†’ generateDice() still runs, but React ignores it, because the state is already managed internally.
  3. Problem: If generateDice() is expensive, you waste CPU calling it unnecessarily on every render.
