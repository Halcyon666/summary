## U30Air SMB共享问题

### 症状

- `\\192.168.0.1\U30 Air` 在 Windows 11 上访问失败
- 资源管理器显示 `Network Error (1208)`
- 同一共享在 Windows 10 上可以正常访问

### 原因

Windows 11 Pro 24H2 及更高版本默认要求 SMB 签名。
匿名或来宾 SMB 访问在需要 SMB 签名时不生效。
ZTE `U30 Air` 共享设备类似于来宾 SMB 设备，因此 Windows 11 会阻止访问，而 Windows 10 仍然允许。

### 解决方案

以管理员身份运行 PowerShell：

```powershell
Set-SmbClientConfiguration -RequireSecuritySignature $false -Force
Set-SmbClientConfiguration -EnableInsecureGuestLogons $true -Force
Get-SmbClientConfiguration | Format-List RequireSecuritySignature,EnableInsecureGuestLogons
net use * /delete /y
```

预期结果：

- `RequireSecuritySignature : False`
- `EnableInsecureGuestLogons : True`

如果本地组策略覆盖了设置，还需检查：

- `计算机配置 -> Windows 设置 -> 安全设置 -> 本地策略 -> 安全选项 -> Microsoft 网络客户端: 始终进行数字签名` = `已禁用`
- `计算机配置 -> 管理模板 -> 网络 -> Lanman 工作站 -> 启用不安全的来宾登录` = `已启用`

### 回滚

```powershell
Set-SmbClientConfiguration -RequireSecuritySignature $true -Force
Set-SmbClientConfiguration -EnableInsecureGuestLogons $false -Force
```

### 备选方案（通过本地组策略）

1. `Win + R` → 输入 `gpedit.msc` → 回车。
2. 路径：
   ```
   计算机配置 → 管理模板 → 网络 → Lanman 工作站
   ```
3. 找到策略：
   **启用不安全的来宾登录 (Enable insecure guest logons)**
4. 双击 → 选择 **已启用** → 确定。
5. 运行 `gpupdate /force` 刷新策略。


## win10系统拷贝文件或者删除文件问题

问题：剩余时间：正在计算  <br/>
处理方案: 使用CMD管理员模式执行 ，然后重启电脑。
```cmd
sfc /scannow
```

执行后的结果：Windows 资源保护找到了损坏文件并成功修复了它们。<br/>
对于联机修复，位于 windir\Logs\CBS\CBS.log 的 CBS 日志文件中<br/>
有详细信息。例如 C:\Windows\Logs\CBS\CBS.log。对于脱机修复，<br/>
/OFFLOGFILE 标记提供的日志文件中有详细信息。

![20250927015301](https://s2.loli.net/2025/09/27/p5vbtsFy1wqANcz.png)

如果还是没有效果，使用WiseForce Deleter删除文件夹。

## win10禁用更新

```cmd
gpedit.msc
```

在“本地组策略编辑器”窗口的左侧面板中，请根据以下路径导航到Windows Update文件夹：“计算机配置”>“管理员模板”>“Windows组件”>“Windows更新”。
更改“删除使用所有windonws更新功能的访问权限”为启用。
更改“配置自动更新为”为禁用。

:::info
打开更新方式 <br/>

更改“配置自动更新为”为启用。
:::
