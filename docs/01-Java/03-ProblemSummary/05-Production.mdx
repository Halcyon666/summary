# 生产问题

## 处理问题思路
- 问题定界，确定是系统内部问题还是，其它系统问题、或者网络原因
- 如果严重先考虑回退(或者重启应用)，先解决掉问题，再找问题的原因
- 原因：连接池问题（线程池中使用）、慢SQL问题、`Full GC` 问题等


## 常见问题处理方案：

- CPU使用率过高：检查是否存在死循环、资源争用等情况，使用性能分析工具定位具体代码位置
- 内存泄漏：通过堆dump分析，找出内存占用大的对象，检查对象的生命周期管理是否合理
- 数据库连接异常：检查连接池配置、网络状态，以及是否存在连接未及时释放的情况
